## 1. RAG คืออะไร

**RAG = Retrieval-Augmented Generation**

เป็นเทคนิคที่ทำให้ AI ตอบคำถามจาก **ข้อมูลของเรา** แทนที่จะตอบจากความรู้ที่ถูกเทรนมาเท่านั้น

### 3 ขั้นตอนของ RAG

```text
┌─────────────────────────────────────────────────────────────────────┐
│                         RAG Pipeline                                │
│                                                                     │
│  1. Retrieval              2. Augmented           3. Generation     │
│  ค้นหาข้อมูลที่เกี่ยวข้อง     ใส่ข้อมูลเข้า prompt    AI สร้างคำตอบ    │
│                                                                     │
│  ┌──────────┐           ┌──────────────┐        ┌──────────────┐   │
│  │ข้อมูลของเรา│──search──▶│  prompt + ข้อมูล │──────▶│  AI สรุปตอบ  │   │
│  │(Vector DB)│           │  ที่ค้นเจอ      │        │  เป็นภาษาคน  │   │
│  └──────────┘           └──────────────┘        └──────────────┘   │
└─────────────────────────────────────────────────────────────────────┘
```

### ตัวอย่างการทำงาน

```text
ผู้ใช้ถาม: "ปั๊ม PTT ในเชียงใหม่มีดีเซลไหม?"

1. Retrieval:  ค้นหาใน Vector Database ด้วย Semantic Search
               → พบ 3 records ที่เกี่ยวข้อง (จาก embedding ที่คล้ายกัน)

2. Augmented:  สร้าง prompt ให้ AI:
               "จากข้อมูลต่อไปนี้ [3 records] จงตอบคำถามผู้ใช้..."

3. Generation: AI อ่านข้อมูลแล้วสรุป:
               "ปั๊ม PTT ในเชียงใหม่มี 3 สาขา โดย 2 สาขายังมีดีเซล
                แต่สาขาแม่ริมรายงานว่าดีเซลหมดเมื่อ 2 ชั่วโมงที่แล้ว"
```

### องค์ประกอบหลักของ RAG

| องค์ประกอบ | คืออะไร | ตัวอย่างเครื่องมือ |
|-----------|--------|------------------|
| **Embedding Model** | แปลงข้อความเป็นตัวเลข (Vector) | OpenAI Embeddings, Ollama, Cohere |
| **Vector Database** | เก็บและค้นหา Vector | ChromaDB, FAISS, Qdrant, Pinecone |
| **Semantic Search** | ค้นหาด้วย "ความหมาย" ไม่ใช่แค่ตัวอักษร | Cosine Similarity, ANN Search |
| **LLM** | สร้างคำตอบจากข้อมูลที่ค้นเจอ | Claude, GPT, Llama |

## 2. MCP คืออะไร

**MCP = Model Context Protocol**

เป็น **โปรโตคอลมาตรฐาน** ที่กำหนดวิธีที่ AI Agent เรียกใช้เครื่องมือ (Tools) ภายนอก เปรียบเหมือน **"ปลั๊กไฟมาตรฐาน"** ที่ AI ทุกตัวเสียบใช้ได้

### MCP ทำงานอย่างไร

```text
┌──────────────┐     MCP Protocol      ┌──────────────────┐
│   AI Agent   │◀──────────────────────▶│    MCP Server    │
│ (Claude, etc)│   1. AI เรียก Tool     │                  │
│              │   2. Server ทำงาน      │  Tools:          │
│              │   3. ส่งผลลัพธ์กลับ     │  - search_fuel() │
│              │                        │  - get_summary() │
└──────────────┘                        └──────────────────┘
```

### MCP ≠ RAG

MCP เป็นแค่ **ท่อส่งข้อมูล** ไม่ได้กำหนดว่าข้างในต้องค้นหาแบบไหน

- MCP + Keyword Search = **ไม่ใช่ RAG** (แค่ค้นหาตัวอักษร)
- MCP + Vector Search = **เป็น RAG** (ค้นหาด้วยความหมาย)

## 3. RAG vs MCP ต่างกันอย่างไร

```text
                    ┌─────────────────────────────────┐
                    │           AI Agent               │
                    └────────┬───────────┬─────────────┘
                             │           │
                    ┌────────▼──┐   ┌────▼────────────┐
                    │   MCP     │   │   RAG Pipeline   │
                    │ (Protocol)│   │   (Technique)    │
                    └───────────┘   └──────────────────┘
                    วิธีเชื่อมต่อ      วิธีค้นหาข้อมูล
                    (ท่อส่งข้อมูล)     (เทคนิคการค้นหา)
```

| หัวข้อ | MCP | RAG |
|-------|-----|-----|
| **คืออะไร** | โปรโตคอล (วิธีสื่อสาร) | เทคนิค (วิธีค้นหา+ตอบ) |
| **เปรียบเหมือน** | สายไฟ / ปลั๊ก | วิธีหาหนังสือในห้องสมุด |
| **ทำหน้าที่** | เชื่อม AI กับ Tools | ค้นหาข้อมูลแล้วให้ AI สรุป |
| **ใช้ร่วมกันได้ไหม** | ได้ — MCP เป็นท่อ, RAG เป็นเทคนิคข้างใน | ได้ — RAG ส่งผ่าน MCP ได้ |
| **ต้องมี Vector DB ไหม** | ไม่จำเป็น | จำเป็น |
| **ต้องมี Embedding ไหม** | ไม่จำเป็น | จำเป็น |

### ใช้ร่วมกัน

```text
MCP + RAG = AI เรียก MCP Tool → Tool ค้นหาใน Vector DB → ส่งผลลัพธ์กลับ AI → AI สรุปตอบ
```

## 4. โปรเจกต์นี้อยู่ตรงไหน

### สิ่งที่โปรเจกต์นี้ทำ

```text
ผู้ใช้ถาม "ดีเซลหมดที่ไหน?"
        ↓
   AI (Claude)
        ↓
   เรียก MCP Tool: search_fuel_status("ดีเซล")    ← MCP ✅
        ↓
   Tool ค้นหาด้วย keyword matching:               ← Keyword Search (ไม่ใช่ RAG)
   if query.lower() in search_text
        ↓
   ส่งผลลัพธ์กลับ AI → AI สรุปตอบ                  ← Generation ✅
```

### สิ่งที่ "เป็น" และ "ยังไม่เป็น"

| องค์ประกอบ | สถานะ | อธิบาย |
|-----------|-------|--------|
| **MCP Server** | ✅ มี | AI เรียกใช้ Tool ได้ผ่านโปรโตคอลมาตรฐาน |
| **MCP Tools** | ✅ มี | `search_fuel_status`, `get_fuel_summary` |
| **AI-Ready Data** | ✅ มี | JSONL, Parquet, Data Card |
| **Keyword Search** | ✅ มี | ค้นหาด้วยการเทียบตัวอักษร (`in` operator) |
| **Embedding** | ❌ ยังไม่มี | ยังไม่ได้แปลงข้อมูลเป็น Vector |
| **Vector Database** | ❌ ยังไม่มี | ยังไม่ได้ใช้ ChromaDB, FAISS ฯลฯ |
| **Semantic Search** | ❌ ยังไม่มี | ยังค้นหาด้วย keyword ไม่ใช่ความหมาย |
| **RAG** | ❌ ยังไม่เต็มรูปแบบ | ขาด Embedding + Vector Search |

### สรุปสั้น ๆ

> โปรเจกต์นี้คือ **MCP Server ที่มี Keyword Search** ไม่ใช่ RAG เต็มรูปแบบ
> แต่โครงสร้างพร้อมต่อยอดเป็น RAG ได้ เพราะข้อมูลเป็น AI-Ready แล้ว

## 5. ถ้าจะทำ RAG จริง ต้องเพิ่มอะไร

### Roadmap: จาก Keyword Search → RAG

```text
ปัจจุบัน                              อนาคต
┌──────────────────┐               ┌──────────────────────────────┐
│  radar_clean.jsonl│               │  radar_clean.jsonl            │
│        ↓          │               │        ↓                      │
│  Keyword Search   │    ──────▶    │  Embedding Model              │
│  (string matching)│               │        ↓                      │
│        ↓          │               │  Vector Database (ChromaDB)   │
│  ส่งผลลัพธ์ให้ AI  │               │        ↓                      │
└──────────────────┘               │  Semantic Search              │
                                    │        ↓                      │
                                    │  ส่งผลลัพธ์ให้ AI              │
                                    └──────────────────────────────┘
```

### สิ่งที่ต้องเพิ่ม (3 ขั้น)

**ขั้น 1: Embedding** — แปลงข้อมูลเป็น Vector
```python
# ตัวอย่าง: ใช้ sentence-transformers (รันในเครื่อง ไม่ต้องใช้ API)
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
text = "ปั๊ม ปตท. สาขาเชียงใหม่ ดีเซลหมด"
vector = model.encode(text)   # → [0.023, -0.041, 0.087, ...] (384 มิติ)
```

**ขั้น 2: Vector Database** — เก็บ Vector ไว้ค้นหา

### Keyword Search vs Semantic Search

| คำถามผู้ใช้ | Keyword Search | Semantic Search (RAG) |
|------------|----------------|----------------------|
| "ดีเซลหมด" | ✅ เจอ (ตรงตัว) | ✅ เจอ |

## 6. สรุปเปรียบเทียบ

```text
┌─────────────────────────────────────────────────────────────┐
│                    ภาพรวมทั้งหมด                              │
│                                                             │
│  ┌─────────┐    ┌───────────┐    ┌────────────┐            │
│  │ AI-Ready │───▶│ MCP Server │───▶│  AI Agent   │            │
│  │  Data    │    │  + Tools   │    │ (Claude etc)│            │
│  └─────────┘    └─────┬─────┘    └────────────┘            │
│                       │                                     │
│              ┌────────┴────────┐                            │
│              │   วิธีค้นหาข้อมูล  │                            │
│              ├─────────────────┤                            │
│              │ Keyword Search  │ ← โปรเจกต์นี้ (ปัจจุบัน)     │
│              │ Semantic Search │ ← RAG (อนาคต)              │
│              └─────────────────┘                            │
└─────────────────────────────────────────────────────────────┘
```

| | Keyword Search (ปัจจุบัน) | RAG (อนาคต) |
|---|---|---|
| **วิธีค้นหา** | เทียบตัวอักษร | เทียบความหมาย (Vector) |
| **ความแม่นยำ** | ต้องพิมพ์ตรง ๆ | เข้าใจบริบท ค้นหาคำใกล้เคียงได้ |
| **ข้ามภาษา** | ไม่ได้ | ได้ (ขึ้นอยู่กับ Embedding Model) |
| **ความซับซ้อน** | ง่าย ไม่ต้อง setup อะไรเพิ่ม | ต้องมี Embedding Model + Vector DB |
| **เหมาะกับ** | ข้อมูลน้อย, ค้นหาตรง ๆ | ข้อมูลเยอะ, คำถามหลากหลาย |
| **โปรเจกต์นี้** | ✅ ใช้อยู่ | ❌ ยังไม่ได้ทำ (แต่พร้อมต่อยอด) |

---

> **Key Takeaway:**
> - **MCP** = วิธีเชื่อมต่อ AI กับ Tools (ท่อส่งข้อมูล)
> - **RAG** = วิธีค้นหาข้อมูลด้วยความหมาย (เทคนิค)
> - **โปรเจกต์นี้** = ใช้ MCP + Keyword Search ซึ่งพร้อมอัปเกรดเป็น MCP + RAG ในอนาคต
> - **ข้อมูล AI-Ready** (JSONL, Parquet) เป็นพื้นฐานที่ดี ไม่ว่าจะใช้วิธีค้นหาแบบไหน
